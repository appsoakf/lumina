对本项目所有功能代码进行 Python 规范审查。

## 执行步骤

1. 使用 Explore agent 收集所有 `.py` 和 `.json` 文件内容（排除模型文件、图片、音频等资源文件）
2. 逐文件审查，覆盖以下 10 个维度：
   - 项目结构：模块划分、职责单一、循环依赖
   - PEP 8 风格：命名、缩进、行长、空行
   - 类型标注：PEP 484 type hints 覆盖度
   - 错误处理：异常粒度、裸 except、资源清理
   - 安全性：硬编码密钥、注入风险、输入校验
   - 并发安全：竞态条件、锁粒度、死锁风险
   - 代码复用：DRY、魔法数字、过度抽象
   - 文档：docstring（PEP 257）、关键逻辑注释
   - 依赖管理：requirements.txt / pyproject.toml 完整性
   - 性能：不必要的阻塞、内存泄漏、资源未释放

## 输出格式

按严重程度分级输出，每条附文件路径和行号：

- 🔴 严重 — 会导致 bug、安全漏洞或数据丢失
- 🟡 建议 — 不影响运行但违反规范或可改进
- 🟢 良好 — 值得保持的设计（仅列出 1-2 条，不做无意义夸奖）

只报告发现的问题，不输出空维度。最后给出一段总结。
